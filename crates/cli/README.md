# sendgrid-cli

An agent-friendly command-line interface to the **SendGrid v3 API** — validated,
safety-gated access to **all 391 operations** across 46 API groups, not a
hand-picked subset. The whole surface is autogenerated from the official
[`twilio/sendgrid-oai`](https://github.com/twilio/sendgrid-oai) OpenAPI specs.

Installs a single binary named `sendgrid`.

## Install

```sh
cargo install sendgrid-cli
```

## Use

```sh
export SENDGRID_API_KEY=SG.xxxx        # preferred over --api-key

# <domain> <subgroup> <verb>-<noun>
sendgrid mail send send-mail --body @msg.json

# Preview the exact HTTP request without sending (global flag — before the subcommand):
sendgrid --dry-run mail send send-mail --body @msg.json

# Serve the bundled MCP server over stdio for an AI agent:
sendgrid mcp
```

Global flags (`--dry-run`, `--output`, `--api-key`, `--page-token`, …) are
accepted before or after the subcommand. `--dry-run` previews the request
(bounded by default; use `--query` for full fidelity). Secrets are always redacted.

## Built for agents

- A small, discoverable command tree generated from the operation registry.
- `--dry-run` + side-effect classes (read / write / destructive / send) so a
  caller can preview and reason about safety before committing.
- The same engine is reachable as an [MCP server](https://crates.io/crates/sendgrid-mcp)
  via `sendgrid mcp`, and as a library via [`sendgrid-core`](https://crates.io/crates/sendgrid-core).
